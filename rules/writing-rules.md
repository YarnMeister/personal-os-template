# Writing Rules · Updated: 2026-06-21

Apply these rules for any writing task: document drafting, stakeholder updates, summaries, email drafts, meeting notes.

## Format

- Use bullet structure for recommendations, action items, and option comparisons.
- Use prose paragraphs for narrative context (background, rationale, executive summaries).
- Maximum 3 levels of nesting. If you need a 4th level, restructure.
- Use Markdown headers (##, ###) as structural dividers — not bold text.

## Length

- Default: as short as possible while complete. Cut filler phrases ("it is worth noting that", "in order to", "it is important to").
- For Confluence documents: include a one-sentence TL;DR at the top.
- For stakeholder updates: max 150 words unless complexity demands more.

## Voice and Tone

- Active voice. "We decided X" not "X was decided."
- Org terminology: use terms as defined in context/org.md glossary. Never use generic alternatives for terms with canonical definitions.
- Avoid hedging language in recommendations. Make a recommendation, then note caveats separately.

## Action Items

- Every action item must have: what, who, by when.
- Format: `[Action] — Owner: [Name] — Due: [Date]`
- Never write an action item without a named owner and date.
