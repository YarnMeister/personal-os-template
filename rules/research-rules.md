# Research Rules · Updated: 2026-06-21

Apply these rules for any research task: discovery queries, source lookups, evidence-gathering.

## Source Hierarchy

1. **Local context files first:** context/active.md, context/org.md, memory/learnings.md, projects/*.md, knowledge/this-week.md
2. **MCP sources second** (when local context is insufficient): Glean → Confluence → Miro (per memory/tools.md routing policy, M3+)
3. **Never fabricate** to fill a gap — surface the gap explicitly to the user.

## Document-Grader Pattern

When using Glean: if fewer than 3 relevant results, escalate to Confluence deep search before concluding the information doesn't exist.

## Citing Sources

- Always state which file or MCP source a claim comes from.
- If synthesising across multiple sources, name all sources.
- If a claim is uncertain, say so. Do not smooth over uncertainty.

## Consolidation

- After a live MCP lookup, write a synthesis to the relevant local file (projects/<name>.md or knowledge/this-week.md) before ending the session. Future queries use the local file, not the live source.
- Never run an MCP query that was already answered in a local file. "Consolidate once, reference often."
